1. Application Definition (Authoring JSON)

The Authoring JSON is the editable application definition created by users in the no-code builder. It contains all metadata required for entities, pages, workflows, navigation, roles, permissions, layout, and components.

1.1 Application Metadata
Field	Type	Description
id	string	Unique identifier for the app
name	string	Human-readable name
version	string	Semantic version, e.g., "1.0.0"
multiTenant	boolean	Whether the app supports multi-tenancy
defaultTenantId	string (optional)	Default tenant ID if single-tenant
createdBy	string	User ID who created the app
createdAt	datetime	Timestamp
1.2 Entity Definition
Entity
 ├── id: string                 // Unique entity name
 ├── displayName: string        // Human-readable name
 ├── tenantScoped: boolean      // Whether the entity is tenant-specific
 ├── properties: Property[]
 ├── relations: Relation[]
 └── calculatedFields: CalculatedField[]
1.2.1 Property Definition
Field	Type	Description
name	string	Unique property name
type	string	"string"
required	boolean	Whether value is mandatory
readonly	boolean	Always read-only
default	any	Default value
length	int	For string fields, max length
precision	int	For decimal fields
scale	int	For decimal fields
expression	string	Optional: expression for calculated value
persist	boolean	Whether calculated field is stored in DB
1.2.2 Relation Definition
Field	Type	Description
name	string	Relation name
type	string	"one-to-one", "one-to-many", "many-to-one", "many-to-many"
target	string	Target entity ID
inverse	string	Property name in target entity that references back
required	boolean	Whether relation is mandatory
tenantScoped	boolean	If relation respects tenantId
1.2.3 Calculated Field Example
{
  "name": "totalAmount",
  "type": "decimal",
  "calculated": true,
  "expression": "entity.amount + entity.tax",
  "persist": true
}
1.3 Persistence Definition
Field	Type	Description
databaseProvider	string	e.g., "postgres"
namingStrategy	string	e.g., "snake_case"
entities	array	List of entity persistence definitions
1.3.1 Entity Persistence
Field	Type	Description
entity	string	Entity ID
tableName	string	Database table name
indexes	array	List of indexes (properties + unique flag)
properties	array	DB-level type overrides
1.4 Page Definition
Page
 ├── id: string
 ├── type: string ("EntityList" | "EntityEdit" | "Custom")
 ├── entity: string (optional)
 ├── layout: LayoutNode[]      // Root node of page layout
 ├── listConfig: ListConfig    // For entity lists
 ├── fieldOverrides: FieldOverride[]
 └── permissions: Permissions
1.4.1 LayoutNode
Field	Type	Description
id	string	Node ID
type	string	"Section", "Row", "Tabs", "Tab", "Field", "DataTable"
props	object	Node-specific properties
children	LayoutNode[]	Nested nodes
1.4.2 Field Override
Field	Type	Description
field	string	Entity property name
readonly	string	Expression to determine readonly at runtime
visible	string	Expression to determine visibility at runtime
default	any	Optional default value
1.4.3 ListConfig
Field	Type	Description
columns	string[]	Properties to show in table
filterFields	string[]	Properties allowed for filtering
quickSearchFields	string[]	Properties searchable in global search
defaultSort	object[]	Field + direction ("asc"/"desc")
pageSize	int	Rows per page
1.5 Navigation Definition
NavigationItem
 ├── id: string
 ├── label: string
 ├── page: string (Page ID)
 ├── children: NavigationItem[]
 └── tenantId: string (optional)
1.6 Workflow Definition (BPMN-aligned)
Field	Type	Description
id	string	Workflow ID
entity	string	Target entity
tenantId	string (optional)	Tenant-specific workflow
startEvent	object	Trigger type: onCreate, onUpdate, onDelete, manual, timer
tasks	Task[]	List of tasks (updateField, createEntity, sendNotification, customAction)
sequenceFlows	SequenceFlow[]	Order of tasks with optional condition expressions
1.7 Roles & Permissions

Role

id : string

name : string

inherits : string[] (optional)

tenantScoped: boolean

Permission

resourceType: "Entity" | "Page" | "Workflow"

resourceId: string

actions: "create" | "read" | "update" | "delete" | "execute"

tenantScoped: boolean

1.8 Component Registry
Field	Type	Description
id	string	Unique component ID
category	string	"field", "layout", "data"
version	string	Semantic version
propsSchema	object	Key-value types for component props
1.9 Themes
Field	Type	Description
id	string	Theme ID
colors	object	primary, secondary, background, text
typography	object	fontFamily, fontSize
2. Release Snapshot
ReleaseSnapshot
 ├── ReleaseId: string
 ├── Version: string
 ├── Metadata: Copy of Authoring JSON
 ├── TenantOverrides: optional
 └── ValidationStatus: "Pending" | "Valid" | "Invalid"

Immutable snapshot of Authoring JSON.

Validates all expressions and layout rules.

Used for deployment and tenant installation.

3. Compiler

Transforms Release Snapshot into optimized runtime artifact.

CompiledRuntimeArtifact
 ├── CompiledPages
 │    ├── RenderTree: flattened layout
 │    ├── StaticNodes: SSR safe nodes
 │    ├── DynamicNodes: expression-driven nodes
 │    └── ExpressionMap: expression ID → expression string
 ├── CompiledWorkflows
 │    ├── Tasks
 │    ├── SequenceFlows
 │    └── StartEvents
 ├── CompiledNavigation: role-specific navigation trees
 ├── DBMigrations: SQL scripts
 └── MetadataVersion
4. Environment Deployment
Environment
 ├── EnvironmentId: string
 ├── TenantId: string
 ├── ReleaseId: string
 ├── EnvironmentType: dev | test | prod
 ├── DBConnection
 ├── RuntimeMetadata: CompiledRuntimeArtifact
 └── DeploymentStatus: Pending | Completed

Runs migrations

Registers workflows

Caches compiled metadata
